# 🏸 배드민턴 운영용 사이트

> **코트 배치 + 참석자 관리 + 참여 횟수 누적**을 한 화면에서 처리하는 정적 웹앱입니다.  
> 동호회/모임 운영 시 매 라운드마다 사람을 배치하고, 게임 종료 시 참여 횟수를 자동 반영할 수 있습니다.

## ✨ 주요 기능

- **코트 중심 운영**
  - 코트를 원하는 만큼 추가/삭제하고, 각 코트는 **최대 4명(복식 기준)** 슬롯으로 구성됩니다.
- **참석자 일괄 등록**
  - 쉼표(`,`) 또는 줄바꿈으로 여러 명을 한 번에 입력할 수 있습니다.
- **자동 추천 배치**
  - 아직 코트에 배치되지 않은 참석자를 우선 대상으로, 빈 슬롯을 자동으로 채웁니다.
- **참여 횟수 관리**
  - 게임 종료 시 코트에 있었던 인원의 참여 횟수를 자동 증가시킵니다.
  - 전체 게임 종료 / 코트별 게임 종료 모두 지원합니다.
  - 참석자 목록에서 `− / ＋` 버튼으로 수동 조정도 가능합니다.
- **지속 저장**
  - 모든 데이터는 브라우저 `localStorage`에 자동 저장됩니다.
  - 새로고침 후에도 상태가 유지됩니다.
- **운영 보조 UI**
  - 실시간 통계(코트 수, 참석자, 배치 완료, 미배치, 빈 자리)
  - 현재 선택된 슬롯 안내 배지
  - 배치된 참석자 시각적 표시

## 🧩 화면 구성

- **상단 액션**
  - `코트 비우기`: 코트 슬롯만 초기화
  - `전체 초기화`: 참석자/코트/참여 횟수 전체 초기화
  - `게임 종료 + 참여횟수 반영`: 전체 코트 반영 후 슬롯 초기화
- **좌측 패널 (코트 편성)**
  - 코트 카드 목록, 슬롯 선택/해제, 코트별 게임 종료
- **우측 패널 (참석자 관리)**
  - 참석자 입력, 참여 횟수 정렬 목록, 수동 증감/삭제

## 🧠 동작 로직 요약

### 1) 정렬 우선순위
참석자 목록은 아래 기준으로 정렬됩니다.

1. 참여 횟수 오름차순 (`count`가 낮은 사람 우선)
2. 미배치 인원 우선 (`isPlaced === false`)
3. 최초 등록 순서

### 2) 배치 규칙
- 참가자를 코트 슬롯에 배치할 때, **동일 이름은 기존 코트에서 자동 해제 후 새 슬롯으로 이동**합니다.
- 즉, 같은 이름이 동시에 여러 슬롯에 존재하지 않도록 처리됩니다.

### 3) 게임 종료 처리
- **전체 게임 종료**: 모든 코트의 현재 인원을 중복 제거 후 카운트 +1, 이후 코트 전체 비움
- **코트별 게임 종료**: 해당 코트 인원만 중복 제거 후 카운트 +1, 해당 코트만 비움

## 🚀 실행 방법

별도 빌드 없이 정적 파일로 동작합니다.

### 방법 A) 파일 직접 열기
1. 저장소 클론
2. `index.html`을 브라우저에서 열기

### 방법 B) 로컬 서버 권장
```bash
python3 -m http.server 8000
```
브라우저에서 `http://localhost:8000` 접속

## ⌨️ 사용 팁

- 참석자 입력창에서 **Ctrl+Enter (macOS: Cmd+Enter)** 로 빠르게 추가할 수 있습니다.
- 코트 슬롯을 먼저 선택한 뒤, 참석자 목록에서 이름을 눌러 배치하세요.
- 운영 중 혼잡할 때는 `자동 추천 배치`로 빈 슬롯을 빠르게 채울 수 있습니다.

## 💾 데이터 저장 형식

앱은 다음 키로 저장합니다.

- `badminton-ops-site-html-v1`

저장 객체 예시:

```json

  "courts": [{ "id": "...", "players": ["A", "B", "", ""] }],
  "participants": ["A", "B", "C"],
  "participationCounts": { "A": 3, "B": 1 }
}
```

## 🔒 참고 사항

- 데이터는 서버가 아닌 **현재 브라우저**에 저장됩니다.
- 브라우저 캐시/스토리지 삭제 시 데이터가 사라질 수 있습니다.
- 동일 이름 참석자를 여러 명 따로 관리하려면 닉네임 구분(예: `민수1`, `민수2`)을 권장합니다.
